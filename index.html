the challenge, the opportunity

Crypto faces a paradox in its value. While the worth of cryptographic consensus and finality has been proven over several critical market cycles, commercial access to crypto assets has been challenged by the inherent risks that make them unique in the first place: they are secured purely by code. Transactions are irreversible and considered correct by default (code-is-law). More ownership with more risk. Economic abstraction is needed.

So we are presented with this challenge and resulting opportunity. To bring these assets to more consumers in ways that don't sacrifice crypto's inherent value, by offering useful automations and safer self-custody with AI and not just more people holding your keys. To allow crypto to expand into everyday transactions with onchain agents. Maintaining the intents of users as full owners, in a holistic, intelligent system, accessible by anyone with an internet connection. Not a number interface: an operating system for money using natural language. This is our premise in constructing ùòïaùòïŒπ for onchain agents.

the solution, as an operating system

To elaborate, we conceive of the onchain agent as an extension of the user in the paradigmatic principal-agent relationship. These agents should understand user intents, and the user should understand their solutions. Preferably, in natural language, both for their instruction and execution. If we can express and map transactional intents to the deterministic outputs of smart contracts like this, we can actually achieve secure guardrails for people to delegate crypto ownership tasks, starting with common transaction routes and simple modules, in the form of commands.

intents engine

To begin, onchain routes to assist users in completing intents are stored in a router smart contract, the Intents Engine (IE). IE simplifies and secures existing agent toolkits for crypto by providing onchain guarantees related to transaction previews and command execution.

For example, the intent to send an asset like a token is sent onchain as a command using conventional syntax, such as send 1 ETH to alice or swap USDC for 1 ETH. The action of "send" is mapped to the relevant contract interface for the given asset (e.g., "ETH"), whether it be ERC20 or native, as well as the address and decimal convention of the asset. The user is guaranteed that their instructions will be delivered exactly as confirmed in simple english. Over time, more function calls will be added to IE, or deployed separately, such as price feeds and other oracles. But for immediate purposes, IE services the common needs for agents and crypto users to get reliable token labels, usernames (through ENS and complimentary namespaces), and prices. From this starting point, a tangible feedback loop of prompting between user and agent can be initiated on ùòïaùòïŒπ to manage their finances faster and more personally than what is possible with traditional services.

smart accounts

Agents provide far more than just an interface to use crypto: they can become active participants in helping plan and program around it. A "smart account" is a wallet that users can run arbitrary code on by using a smart contract. This customizability makes smart accounts more secure and useful than ordinary wallets, or EOAs. Where EOAs offer massive accessibility to improve against legacy options (anyone online can get onboarded), smart accounts improve on EOAs by matching and further improving upon the UX of banking by allowing applications to be built around self-custody (such as hybrid custody arrangements where control is shared or governed). They do this by providing an open canvas against which users can program access to their own money.

Therefore, to benefit most from agents, crypto users should employ a smart account and customize it to fit their security needs and frequent commands. Future-looking orders and permissions, such as the right to spend certain amounts, or to call certain contracts, can also be engaged with smart accounts by using modules and other add-ons curated and offered through ùòïaùòïŒπ.

Solutions for intents can be judged on objective criteria, such as gas efficiency and output amounts. The more subjective aspects of managing crypto assets, such as labelling and safety parameters, are frequently open-ended and difficult to add to deterministic workflows familiar to smart contracts. the future, more or less, as we know it

In five years, we expect that most financial transactions will be mediated by agents running on large open language (LOL) models, settled on blockchains, secured by cryptography and economic incentives provided by crypto assets. This just tracks with the internet so far in our modest opinion. In our estimation, too, language is the natural interface for this kind of more personal ownership, and these kinds of agents, if we are to replicate and improve on how we direct our own affairs and finance. Simplicity feels underrepresented in this domain. So we will proceed with this onchain agent infra: NSA, LOL, and TIE, and build on what works.
